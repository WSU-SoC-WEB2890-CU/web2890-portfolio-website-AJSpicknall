Case Study: Campus Event Hub

A UX-focused redesign that helped students quickly find events matching their interests, schedules, and campus location.

Project Overview

Role: UX designer and front-end developer

Timeline: 8 weeks

Goal: Reduce friction in event discovery and increase event attendance among first-year students.

Problem Statement

Students reported that event details were scattered across social channels, posters, and disconnected web pages. Most users missed time-sensitive events because they could not filter by date, category, and location quickly.

Research + Insights

  • Interviewed 10 students from different majors and class years.
  • Ran a competitive analysis of 4 event directory interfaces.
  • Found that mobile filtering and calendar sync were top priorities.

Design Process

  1. Mapped user flows for browse, filter, and save-event tasks.
  2. Created low-fidelity wireframes focused on clear information hierarchy.
  3. Built an interactive prototype and tested with 6 users.
  4. Updated layout, labels, and filter behavior based on test feedback.

Accessibility Decisions

  • Improved contrast beyond WCAG AA target for body and action text.
  • Added strong focus indicators and logical keyboard traversal.
  • Used semantic landmarks for easier screen reader navigation.

Outcome

Usability testing after iteration showed a 39% reduction in time to find a relevant event and complete a calendar-save action. Participants described the final version as easier to scan and less overwhelming on mobile.

Reflection

This project reinforced the value of pairing visual design with clear task flows. The strongest improvements came from testing early and changing interaction details before full development.
